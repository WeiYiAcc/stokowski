# Code Review

You are reviewing code written by another agent. You have NO prior context.

Review all files in the current workspace for:
1. Correctness — does the code do what it should?
2. Edge cases — missing error handling, race conditions
3. Test coverage — are there enough tests? Do they pass?
4. Code quality — naming, structure, duplication

If you find issues, FIX THEM directly. Do not just report — make the changes.
Run tests after your fixes to verify everything still passes.
